## Creating Content {#creating}

Start creating professional documentation with WIZIUM's intuitive tools. From quick notes to comprehensive guides, our editor adapts to your needs.

<Steps>
  <Step title="Create New Document" icon="file-plus">
    Click the "New Document" button and choose a template or start blank.

    <Expandable title="Document Templates" default-open="false">
      Choose from API docs, user guides, release notes, or create custom templates.
    </Expandable>
  </Step>
  <Step title="Write Content" icon="edit">
    Use markdown or rich text formatting to create engaging content.

    <CodeGroup tabs="Markdown,Richtext">
      ````markdown
      ## API Authentication

      To authenticate, include your API key in the header:

      ```
      Authorization: Bearer YOUR_API_KEY
      ```
      ````
      ````html
      <h2>API Authentication</h2>
      <p>To authenticate, include your API key in the header:</p>
      <pre><code>Authorization: Bearer YOUR_API_KEY</code></pre>
      ````
    </CodeGroup>
  </Step>
  <Step title="Add Media" icon="image">
    Embed images, videos, and diagrams to enhance your documentation.

    <Callout kind="info">Supported formats include PNG, JPG, GIF, MP4, and Mermaid diagrams.</Callout>
  </Step>
  <Step title="Publish" icon="send">
    Set visibility permissions and publish your document to your team.
  </Step>
</Steps>

## Organizing Documentation {#organizing}

Keep your documentation structured and discoverable with effective organization strategies.

<Tabs>
  <Tab title="Folders" icon="folder">
    Create hierarchical folder structures for different projects or topics.

    | Folder Type | Example |
    |-------------|---------|
    | By Project | /Project Alpha, /Project Beta |
    | By Function | /API Docs, /User Guides |
    | By Team | /Engineering, /Product, /Support |
  </Tab>
  <Tab title="Tags" icon="tag">
    Use tags for cross-cutting organization and flexible filtering.

    <Expandable title="Tag Best Practices" default-open="false">
      - Use consistent naming conventions
      - Create tag hierarchies (api/v1, api/v2)
      - Limit to 5-10 tags per document
      - Review and clean up unused tags regularly
    </Expandable>
  </Tab>
  <Tab title="Collections" icon="book-open">
    Group related documents into collections for guided reading experiences.
  </Tab>
</Tabs>

## Search and Discovery

Find information quickly with WIZIUM's powerful search capabilities.

<Steps>
  <Step title="Basic Search" icon="search">

  </Step>
  <Step title="Advanced Filters" icon="filter">

## Version Management

Track changes and maintain document history with version control.

<CodeGroup tabs="Compare Versions,Restore Version">
  ````javascript
  // Compare versions side-by-side
  // In document view: History > Compare
  // Select two versions to diff
  ````
  ````bash
  # Restore previous version via API
  curl -X POST "https://api.wizium.com/v1/documents/doc_123/restore" \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -d '{"version_id": "v1.2"}'
  ````
</CodeGroup>

<Callout kind="success">Regularly review version history to understand how your documentation evolves over time.</Callout>

## Productivity Tips

Boost your efficiency with these proven techniques:

- Use keyboard shortcuts for common actions (Ctrl+S to save, Ctrl+K for search)
- Create document templates for consistent formatting
- Set up email digests for documentation updates
- Use the "Recently Viewed" section for quick access
- Enable dark mode for extended editing sessions

<Expandable title="Advanced Tips" default-open="false">
  - Master search operators for precise queries
  - Use document analytics to identify popular content
  - Set up personal dashboards for customized views
  - Leverage API access for custom integrations
</Expandable>
